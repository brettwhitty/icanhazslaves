# draft

## Description
The `draft` action type involves creating or modifying non-executable technical artifacts, such as templates, SOPs, implementation plans, and architectural overview documents.

## Scope
- Authoring new `.md` files in `docs/` or `state/`.
- Proposing implementation plans.
- Creating walkthroughs or reports.
- Drafting mode specification templates.

## Allowed Tools (Heuristic)
- `write_to_file`
- `replace_file_content`
- `multi_replace_file_content`
- `generate_image` (for UI mockups or diagrams)
